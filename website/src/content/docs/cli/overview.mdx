The Imbrace CLI (`@imbrace/cli`) is a terminal tool for interacting with the Imbrace platform. Commands call the Imbrace platform directly via the `@imbrace/sdk` package. Source: [imbraceltd/imbrace-cli](https://github.com/imbraceltd/imbrace-cli).

```
imbrace (CLI) ‚Üí https://app-gatewayv2.imbrace.co (Imbrace Platform)
```

## Authentication Flow

Credentials live inside **named profiles** (AWS-style). One CLI install can hold multiple accounts/environments ‚Äî switch via `imbrace profile use <name>`, `--profile <name>` flag, or `IMBRACE_PROFILE` env var. Each profile maps 1:1 to the SDK `ImbraceClientConfig`.

The CLI supports two authentication methods (both written into a profile):

**1. API Key** (credential starts with `sk-` or `api_`):
- Passed directly to SDK: `new ImbraceClient({ apiKey: credential, env, baseUrl, organizationId, ... })`
- Stored as-is in the target profile

**2. Email + Password** (returns JWT token):
- SDK: `client.login(email, password)` ‚Üí token stored in the profile
- JWT token stored as credential

See [Commands ‚Üí Profile](/cli/commands/#profile) for the full profile management surface, including `--base-url` for self-hosted Imbrace instances.

### Credential Storage

Credentials are stored via the `conf` npm package with `projectName: "imbrace"`.

| OS | Config path |
|---|---|
| macOS | `~/Library/Preferences/imbrace-nodejs/config.json` |
| Linux | `~/.config/imbrace-nodejs/config.json` |
| Windows | `%APPDATA%\imbrace-nodejs\Config\config.json` |

Stored fields: `credential`, `method` (`"api-key"` | `"password"`), `email`.

### Auto-login

Every CLI command (except `login`) extends `BaseCommand`, which calls `ensureLoggedIn()` on init. If no credential is found, it interactively prompts the user to choose a login method (API Key or Email + Password).

<Aside type="tip">
  All commands accept `--json` for machine-readable output and `-h` / `--help` for usage.
</Aside>

## Workflow Node Architecture

Workflows use a **linked list** structure via `nextAction` pointers:

```typescript
{
  version: {
    trigger: {
      name: "trigger",
      type: "PIECE_TRIGGER" | "EMPTY" | "PIECE" | "ROUTER" | "LOOP_ON_ITEMS" | "CODE",
      settings: { pieceName, pieceVersion, input, ... },
      nextAction: { ... },
      children?: any[],          // ROUTER branches
      firstLoopAction?: any,     // LOOP_ON_ITEMS body
    }
  }
}
```

All node operations use `applyFlowOperation()` with types: `UPDATE_TRIGGER`, `ADD_ACTION`, `UPDATE_ACTION`, `DELETE_ACTION`, `LOCK_AND_PUBLISH`, `CHANGE_STATUS`, `CHANGE_FOLDER`.
