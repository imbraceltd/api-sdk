---
title: TypeScript SDK — 錯誤處理
description: 錯誤類型、自動重試行為以及 Imbrace TypeScript SDK 的最佳實踐。
---

SDK 拋出的所有錯誤都繼承自一個公共基底類別，並具有完整型別，可透過 `instanceof` 精確處理。

### 錯誤層級

```
Error
└── ImbraceError          （基底類別 — 捕獲所有 SDK 錯誤）
    ├── AuthError          （401、403 — 無效或過期的憑證）
    ├── ApiError           （4xx/5xx — 伺服器拒絕請求）
    └── NetworkError       （逾時、連線被拒絕、DNS 失敗）
```

---

### AuthError

伺服器回傳 **401** 或 **403** 時拋出。

```typescript
import { AuthError } from "@imbrace/sdk";

try {
  const me = await client.platform.getMe();
} catch (e) {
  if (e instanceof AuthError) {
    // API Key 過期、令牌被撤銷或憑證錯誤
    console.error("需要重新認證：", e.message);
  }
}
```

:::caution
`AuthError` **永遠不會被重試**。SDK 在收到 401/403 時立即拋出 — 需要先修復憑證再重試。
:::

---

### ApiError

針對所有其他 **4xx 和 5xx**（429/5xx 重試耗盡後）拋出。

```typescript
import { ApiError } from "@imbrace/sdk";

try {
  await client.marketplace.getProduct("not_exist_id");
} catch (e) {
  if (e instanceof ApiError) {
    console.error(`HTTP ${e.statusCode}: ${e.message}`);
    // 例如："HTTP 404: Product not found"
  }
}
```

| 屬性         | 型別     | 說明                     |
| ------------ | -------- | ------------------------ |
| `statusCode` | `number` | HTTP 狀態碼              |
| `message`    | `string` | 伺服器回應中的錯誤訊息   |

---

### NetworkError

請求未能到達伺服器時拋出 — 逾時、DNS 失敗或連線被拒絕。

```typescript
import { NetworkError } from "@imbrace/sdk";

try {
  await client.platform.getMe();
} catch (e) {
  if (e instanceof NetworkError) {
    console.error("無法連線 baseUrl：", e.message);
    // 例如："Request timed out after 30000ms"
  }
}
```

### 捕獲所有 SDK 錯誤

```typescript
import { ImbraceError, AuthError, ApiError, NetworkError } from "@imbrace/sdk";

try {
  await client.platform.getMe();
} catch (e) {
  if (e instanceof AuthError) return handleAuthError(e);
  if (e instanceof ApiError) return handleApiError(e);
  if (e instanceof NetworkError) return handleNetworkError(e);
  if (e instanceof ImbraceError) return handleUnknownError(e);
  throw e; // 重新拋出非 SDK 錯誤
}
```

### 自動重試行為

`HttpTransport` 自動對臨時錯誤進行重試：

| 條件                        | 行為                                     |
| --------------------------- | ---------------------------------------- |
| HTTP **429**（頻率限制）    | 最多重試 2 次                            |
| HTTP **5xx**（伺服器錯誤）  | 最多重試 2 次                            |
| 網路錯誤 / 逾時             | 最多重試 2 次                            |
| HTTP **401 / 403**          | 不重試 — 立即拋出 `AuthError`            |
| HTTP **4xx**（其他）        | 不重試 — 立即拋出 `ApiError`             |

**退避公式：** 重試間隔 `2^retryCount` 秒（2s → 4s）。

最壞情況：3 次嘗試（1 次初始 + 2 次重試）後拋出例外。

### 取消請求

使用標準 `AbortController` 取消進行中的請求：

```typescript
const controller = new AbortController();

// 5 秒後取消
setTimeout(() => controller.abort(), 5000);

try {
  const result = await client.marketplace.listProducts(
    { page: 1 },
    { signal: controller.signal },
  );
} catch (e) {
  if (e instanceof NetworkError && e.message.includes("aborted")) {
    console.log("請求已被取消");
  }
}
```

### 最佳實踐

```typescript
// 1. 單獨處理 AuthError — 憑證需要刷新
// 2. 記錄 ApiError.statusCode 以便除錯 — 400=參數錯誤，404=未找到
// 3. 在入口點用 try/catch 包裝 — 不要讓 NetworkError 導致應用程式崩潰
// 4. AuthError 時不要重試 — 在修復憑證之前重試無效

async function safeGetMe(client: ImbraceClient) {
  try {
    return await client.platform.getMe();
  } catch (e) {
    if (e instanceof AuthError) {
      await refreshCredentials();
      return await client.platform.getMe(); // 刷新後重試一次
    }
    throw e;
  }
}
```
