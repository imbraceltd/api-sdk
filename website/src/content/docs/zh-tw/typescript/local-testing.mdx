title: TypeScript SDK — 本地測試

---

本指南允許測試**已建置的套件**（`dist/`），方式與消費者從 npm 安裝完全相同 — 而非原始源碼。

### 前置條件

- Node 18+
- 如需從 Ansible 解密環境變數，需配置 AWS credentials（profile `imbrace`）

### 初始設定（一次性）

#### 1. 建置 SDK

```bash
cd ts
npm install
npm run build
```

#### 2. 全域連結套件

```bash
# 在 ts/ 目錄中
npm link
```

此命令使 `@imbrace/sdk` 可用，就像從 npm 安裝一樣。

#### 3. 連結到測試目錄

```bash
cd ts/tests/local
npm link @imbrace/sdk
```

#### 4. 配置憑證

```bash
cp .env.example .env
```

然後填寫 `.env`。對於開發環境，最快的方式是從 Ansible 提取變數：

```bash
# 從 ts/ 根目錄（或包含 secrets.enc.env 的位置）
AWS_PROFILE=imbrace sops -d ansible/dev/secrets.enc.env | grep IMBRACE >> tests/local/.env
```

實時呼叫所需的最低配置：

| 變數                      | 取得方式                                                                                    |
| ------------------------- | ------------------------------------------------------------------------------------------ |
| `IMBRACE_API_KEY`         | Imbrace Portal，或使用現有 access token 呼叫 `POST /private/backend/v1/third_party_token`  |
| `IMBRACE_ORGANIZATION_ID` | Portal 中的組織 UUID                                                                       |
| `IMBRACE_GATEWAY_URL`     | 留空則使用 `app-gateway.dev.imbrace.co`                                                    |

### 執行測試

```bash
cd ts/tests/local
node test-local.mjs
```

**未提供憑證** — 僅執行實例化 + 資源介面檢查（無網路請求）。

**設置了 `IMBRACE_API_KEY`** — 對 `IMBRACE_GATEWAY_URL` 中的閘道執行完整實時 API 檢查。

### 切換環境

修改 `.env`（或內聯匯出）：

| 環境       | `IMBRACE_GATEWAY_URL`                    |
| ---------- | ---------------------------------------- |
| develop    | `https://app-gateway.dev.imbrace.co`     |
| sandbox    | `https://app-gateway.sandbox.imbrace.co` |
| stable     | `https://app-gatewayv2.imbrace.co`       |

```bash
IMBRACE_GATEWAY_URL=https://app-gateway.sandbox.imbrace.co node test-local.mjs
```

### SDK 變更的迭代流程

每次修改源碼後，重新建置即可使用新的 dist：

```bash
# ts/
npm run build

# tests/local
node test-local.mjs
```

或在獨立終端機中保持編譯器監視模式：

```bash
# 終端機 1 — ts/
npm run dev

# 終端機 2 — ts/tests/local
node test-local.mjs   # 隨時重新執行
```

### 疑難排解

**`Cannot find package '@imbrace/sdk'`**

在 `tests/local/` 內重新執行 `npm link @imbrace/sdk`。若刪除並重新安裝 `ts/` 中的 `node_modules`，連結可能會斷開。

**`ERR_MODULE_NOT_FOUND`（dist 檔案）**

套件尚未建置，或新增了新源碼檔案但未重新建置。在 `ts/` 中執行 `npm run build`。

**實時呼叫回傳 401 / 403**

`IMBRACE_API_KEY` 已過期或無效。生成新金鑰：

```bash
curl -X POST https://app-gateway.dev.imbrace.co/private/backend/v1/third_party_token \
  -H "x-access-token: <your_existing_token>" \
  -H "Content-Type: application/json" \
  -d '{"expirationDays": 30}'
```
