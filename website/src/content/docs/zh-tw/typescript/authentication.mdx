---
title: TypeScript SDK — 身份驗證
description: Imbrace TypeScript SDK 中的 API Key 和 Access Token。
---

SDK 支援兩種認證策略。初始化 `ImbraceClient` 時傳入 `apiKey` 或 `accessToken` — 傳輸層自動處理請求標頭。

### 認證策略

| 策略             | 使用場景                              | 發送的請求標頭            |
| ---------------- | ------------------------------------- | ------------------------- |
| **API Key**      | 後端腳本、排程任務、伺服器端呼叫      | `x-api-key: sk-xxx...`    |
| **Access Token** | 使用者登入後（OTP 或密碼）            | `x-access-token: acc_...` |

---

### API Key（伺服器端）

```typescript
import { ImbraceClient } from "@imbrace/sdk";

const client = new ImbraceClient({
  apiKey: process.env.IMBRACE_API_KEY,
});
```

---

### Access Token（客戶端）

使用者認證後使用：

```typescript
const client = new ImbraceClient({
  accessToken: "acc_xxxxxxxxxxxxx",
});
```

---

### OTP 登入流程

面向使用者應用的推薦方式 — 登入成功後 SDK 自動儲存令牌。

```typescript
import { ImbraceClient } from "@imbrace/sdk";

const client = new ImbraceClient();

// 步驟 1：向使用者信箱發送 OTP
await client.requestOtp("user@example.com");

// 步驟 2：使用者輸入 OTP — 令牌自動儲存
await client.loginWithOtp("user@example.com", "123456");

// 步驟 3：後續所有呼叫均已通過認證
const me = await client.platform.getMe();
```

也支援信箱 + 密碼登入：

```typescript
await client.login("user@example.com", "password123");
```

---

### 令牌管理

```typescript
// 替換令牌（例如 OAuth 刷新後）
client.setAccessToken("new_token...");

// 清除令牌（例如登出時）
client.clearAccessToken();
```

---

### 配置優先級

1. **明確的建構子選項** — `new ImbraceClient({ apiKey: "..." })`
2. **環境變數** — `IMBRACE_API_KEY`、`IMBRACE_GATEWAY_URL`
3. **預設值** — `env: "stable"`，閘道 `https://app-gatewayv2.imbrace.co`
