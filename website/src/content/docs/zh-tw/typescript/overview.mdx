`@imbrace/sdk` 是 Imbrace Gateway 的官方 TypeScript/JavaScript 客戶端，為所有 Imbrace 服務提供完全類型化的存取，支援伺服器端（Node.js）和客戶端（瀏覽器）。

### 核心特性

| 特性                     | 說明                                                                 |
| ------------------------ | -------------------------------------------------------------------- |
| **完整類型安全**          | 100% TypeScript — 所有請求和回應透過 `types/` 完全類型化             |
| **雙重認證**              | API Key（伺服器對伺服器）或 Access Token（JWT / legacy token）        |
| **自動重試**              | 429 和 5xx 自動重試最多 2 次，使用指數退避                           |
| **請求取消**              | 支援 AbortSignal 取消進行中的請求                                    |
| **多環境**                | `develop` / `sandbox` / `stable` 預設，支援按服務覆蓋 URL            |
| **AI 串流輸出**           | 基於 AsyncGenerator 的 AI 補全串流傳輸                               |

### 工作原理

```
Your Code
   │
   ▼
ImbraceClient          ← 唯一入口，連接所有資源
   ├── TokenManager    ← 執行緒安全令牌儲存（OTP / JWT / API Key）
   └── HttpTransport   ← 認證標頭、重試、逾時、錯誤映射
         │
         ▼
   Imbrace Gateway     ← app-gatewayv2.imbrace.co（+ 按服務 URL）
```

`ImbraceClient` 建立一個 `HttpTransport` 並在所有資源類別間共享。每個資源（`client.marketplace`、`client.platform` 等）是輕薄封裝，只負責建構正確的 URL 並呼叫 `transport.request()`。

### 內部架構

| 層級             | 檔案                    | 職責                                               |
| ---------------- | ----------------------- | -------------------------------------------------- |
| 入口點            | `client.ts`             | 初始化所有資源，處理登入輔助函式                    |
| HTTP 傳輸         | `http.ts`               | 認證標頭、重試邏輯、逾時、錯誤映射                  |
| 令牌管理器        | `auth/token-manager.ts` | 儲存和取得 access token                             |
| 服務登錄表        | `service-registry.ts`   | 從環境預設解析按服務 URL                            |
| 資源              | `resources/*.ts`        | 各領域 API 方法封裝（29 個資源）                   |
| 型別              | `types/*.ts`            | 所有 API 形狀的 TypeScript 介面                     |
| 錯誤              | `errors.ts`             | 定義 `AuthError`、`ApiError`、`NetworkError`        |

### 適用場景

| 場景                                    | 推薦認證方式                                |
| --------------------------------------- | ------------------------------------------- |
| Node.js 後端、排程任務、資料管道        | `apiKey` → 請求標頭 `x-api-key`            |
| OTP/OAuth 登入後的前端                  | `accessToken` JWT → `Authorization: Bearer` |
| 使用不透明令牌的 legacy 整合            | `accessToken` opaque → `x-access-token`     |

:::tip
伺服器端使用 `apiKey`。面向使用者的應用使用 OTP，讓 SDK 自動管理令牌。
:::
