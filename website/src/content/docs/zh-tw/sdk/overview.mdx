Imbrace SDK 是 Imbrace Gateway 的官方客戶端，提供 **TypeScript** 與 **Python** 兩種版本。兩個 SDK 皆封裝相同的 Gateway API、相同的資源命名空間、相同的驗證模型以及相同的重試/錯誤語意 — 選擇符合你技術棧的語言即可。

> 在此網站任一處切換語言分頁一次，其餘文件將記住你的選擇。

### 主要功能

| 功能 | 說明 |
|---|---|
| **型別安全** | 每個資源皆提供 TypeScript 型別與 Python 型別提示 |
| **兩種憑證類型** | `apiKey` 或 `accessToken` — 請參閱[驗證](/sdk/authentication/) |
| **自動重試** | 429 與 5xx 狀態碼使用指數退避自動重試，無需額外設定 |
| **串流 AI** | SSE / 非同步迭代器支援 `streamChat` 與 AI 完成 |
| **同步與非同步 (Python)** | `ImbraceClient`（同步）與 `AsyncImbraceClient`（非同步） |
| **取消請求 (TS)** | 使用 `AbortSignal` 傳遞進行中的請求取消 |

### Hello, world

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```typescript
    import { ImbraceClient } from "@imbrace/sdk"

    const client = new ImbraceClient({ accessToken: "acc_your_token" })
    const me = await client.platform.getMe()
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    from imbrace import ImbraceClient

    with ImbraceClient(access_token="acc_your_token") as client:
        me = client.platform.get_me()
    ```
  </TabItem>
</Tabs>

### 可用資源

每個命名空間在兩個 SDK 中皆可使用。方法遵循各語言的慣例 — TypeScript 為 `client.aiAgent.streamChat()`，Python 為 `client.ai_agent.stream_chat()`。

| 命名空間 | 用途 |
|---|---|
| `client.aiAgent` / `client.ai_agent` | 串流 AI 聊天、嵌入向量、Parquet、聊天客戶端子 API |
| `client.chatAi` / `client.chat_ai` | 助理 CRUD（建立/更新/刪除/列出助理） |
| `client.workflows` | 工作流程自動化 — 流程、觸發器、執行紀錄 |
| `client.boards` | CRM 看板 — CRUD、項目、欄位、搜尋、區段、CSV；KH 資料夾與檔案 |
| `client.platform` | 使用者、組織、權限 |
| `client.contacts`、`client.conversations`、`client.messages`、`client.channel` | 聯絡人/頻道層 |
| `client.ai` | 相容 OpenAI 的完成與嵌入向量 |

完整列表與方法參考請參閱[資源](/sdk/resources/)。

### 何時選擇哪種憑證

| | API Key | Access Token |
|---|---|---|
| **Imbrace 是誰的後端？** | 是*你*後端中的一個功能 | *Imbrace 就是*你的後端 |
| **使用者是誰的？** | 你的 | Imbrace 的 |
| **最適合用於** | 伺服器對伺服器、內部腳本、CRM 整合 | 每個終端使用者登入的面向使用者應用程式 |

完整決策樹：[驗證 →](/sdk/authentication/)。
