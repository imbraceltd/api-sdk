SDK 支援兩種憑證類型。將 `apiKey` / `api_key` 或 `accessToken` / `access_token` 傳遞給客戶端建構函式即可 — 傳輸層會自動處理標頭。

### 我該使用哪種憑證？

選擇取決於 **Imbrace 在你的產品中扮演的角色**：

#### 建立在 Imbrace 上 → 使用 Access Token

Imbrace 就是你的後端。你的終端使用者登入 Imbrace（透過 OTP 或密碼）；他們的 `acc_...` 存取權杖即為 SDK 每次請求所使用的憑證。Imbrace 的驗證、資料庫與商業邏輯（助理、知識庫、工作流程、AI 代理）皆以實際登入使用者的身份執行。

典型產品：聊天元件、儀表板或行動應用程式，其中每個使用者的身份即為 Imbrace 使用者。Imbrace 會記錄每位使用者的歷史記錄、權限與稽核軌跡。

#### 包裝 Imbrace → 使用 API Key

你有自己的後端、自己的使用者、自己的資料庫。Imbrace 是你的服務所調用的一個功能 — 「使用 Imbrace 的 AI 來回答這個問題」— 與你執行其他所有操作並行。你的終端使用者永遠不會看到 Imbrace；你的服務使用單一組織層級的 `api_...` 金鑰（由 Imbrace 組織管理員核發）來呼叫 Imbrace。Imbrace 只會看到一個身份（API Key 的服務使用者），而你需要自行處理使用者層級的歸因。

典型產品：現有的 CRM、工單系統或內部工具，將 Imbrace 嵌入為一項功能。一個憑證存放在你的環境變數檔案中，為每個請求提供服務。

| | 建立在 Imbrace 上 (Access Token) | 包裝 Imbrace (API Key) |
| --- | --- | --- |
| **誰負責驗證？** | Imbrace（OTP / 密碼登入） | 你（你自己的驗證） |
| **使用者是誰的？** | Imbrace 使用者 | 你的使用者 |
| **誰的資料庫是權威來源？** | Imbrace 的 | 你的 |
| **Imbrace 看到的身份** | 實際的終端使用者 | 一個服務帳號 |
| **每個使用者的歸因** | 內建 | 由你自行處理 |
| **憑證有效期** | 工作階段範圍；可按需重新整理 | 長效；不會自行過期 |
| **傳送的標頭** | `x-access-token: acc_...` | `x-api-key: api_...` |

> 兩種憑證類型都是一級支援。大多數資源接受任一類型。少數依賴使用者上下文的功能（文件產物、每個使用者的聊天歷史）只有在使用 Access Token 時才有意義。

### 標頭參考

| 憑證 | 傳送的標頭 |
| --- | --- |
| `apiKey` / `api_key` | `x-api-key: api_xxx...` |
| `accessToken` / `access_token` | `x-access-token: acc_xxx...` |

組織上下文已**編碼在憑證本身內部** — 每個 API Key 和每個 Access Token 都只綁定一個組織。Gateway 會根據你傳送的憑證在每次請求時解析組織。你無需將 `organizationId` / `organization_id` 傳遞給 SDK。

關於憑設設定的逐步說明（環境變數、dotenv、密鑰），請參閱[設定指南](/getting-started/setup/#configure-credentials)。

### OTP 登入流程

使用此流程透過電子郵件 OTP 驗證使用者並取得工作階段權杖。登入後 SDK 會自動儲存權杖。

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```typescript
    import { ImbraceClient } from "@imbrace/sdk"

    const client = new ImbraceClient({
      baseUrl: "https://app-gatewayv2.imbrace.co",
    })

    // 步驟 1：發送 OTP 至使用者的電子郵件
    await client.requestOtp("user@example.com")

    // 步驟 2：使用者提交收到的 OTP
    const loginRes = await client.loginWithOtp("user@example.com", "ABC123")
    // loginRes 包含一個短期有效的 login_acc_... 權杖（內部儲存）

    // 步驟 3：兌換成長期有效的存取權杖
    const { token, refresh_token } = await client.auth.exchangeAccessToken("org_your_org_id")

    // 步驟 4：啟用權杖供後續所有呼叫使用
    client.setAccessToken(token)

    // 現在可以使用任何資源
    const { data: boards } = await client.boards.list()
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    from imbrace import ImbraceClient

    client = ImbraceClient()

    # 步驟 1：發送 OTP 至使用者的電子郵件
    client.request_otp("user@example.com")

    # 步驟 2：使用者輸入 OTP — 權杖自動儲存
    client.login_with_otp("user@example.com", "123456")

    # 步驟 3：後續所有呼叫皆已驗證
    me = client.platform.get_me()
    ```

    非同步版本：

    ```python
    from imbrace import AsyncImbraceClient

    async with AsyncImbraceClient() as client:
        await client.request_otp("user@example.com")
        await client.login_with_otp("user@example.com", "123456")
        me = await client.platform.get_me()
    ```
  </TabItem>
</Tabs>

### 情境管理器（僅限 Python）

始終將 Python 客戶端包裝在情境管理器中，以便底層的 `httpx` 連線池能被正確關閉：

```python
# 同步
with ImbraceClient(api_key="api_xxx") as client:
    me = client.platform.get_me()

# 非同步
async with AsyncImbraceClient(api_key="api_xxx") as client:
    me = await client.platform.get_me()
```
