本指南協助你測試**已建置的套件**，完全如同消費者安裝時的方式 — 而非原始原始碼。在發布新版本或重現消費者回報的錯誤時使用。

## 前置需求

- TypeScript：Node 18+
- Python：Python 3.10+ 與 `pip`
- 已設定的 AWS 憑證（`imbrace` 設定檔），如需從 Ansible 解密環境變數

## 環境 URL

兩套 SDK 共用同一組 Gateway 基礎 URL。在你的測試環境中設定一次即可：

| 環境 | `IMBRACE_BASE_URL` / `IMBRACE_GATEWAY_URL` |
| ----------- | ------------------------------------------ |
| develop     | `https://app-gateway.dev.imbrace.co`       |
| sandbox     | `https://app-gateway.sandbox.imbrace.co`   |
| stable      | `https://app-gatewayv2.imbrace.co`         |

## 憑證

從 Ansible 提取即時憑證（或從入口網站貼上）：

```bash
# 從 SDK 儲存庫根目錄
AWS_PROFILE=imbrace sops -d ansible/dev/secrets.enc.env | grep IMBRACE >> /tmp/imbrace.env
```

即時呼叫所需的最低要求：

| 變數 | 取得方式 |
| ----------------- | --------------------------------------------------------------------------------------------- |
| `IMBRACE_API_KEY` | Imbrace 入口網站，或使用現有 Access Token 呼叫 `POST /private/backend/v1/third_party_token` |
| `IMBRACE_BASE_URL` | 上表 URL 之一（未設定時預設為 dev） |

組織上下文已編碼於 API Key 內部 — 你無需傳遞組織 ID。

## TypeScript — 連結已建置的 dist

### 一次性設定

```bash
cd ts
npm install
npm run build
npm link        # 全域暴露 @imbrace/sdk
```

然後在測試資料夾中：

```bash
cd ts/tests/local
npm link @imbrace/sdk
cp .env.example .env
# 填入 .env
```

### 執行測試

```bash
cd ts/tests/local
node test-local.mjs
```

- **沒有憑證** — 僅執行實例化 + 資源表面檢查（無網路連線）。
- **設定 `IMBRACE_API_KEY`** — 針對 `IMBRACE_BASE_URL` 中的 Gateway 執行所有即時 API 檢查。

### 迭代 SDK 變更

每次編輯都需要重新建置，連結才能取用最新變更：

```bash
# 終端機 1 — ts/
npm run dev          # tsc --watch

# 終端機 2 — ts/tests/local
node test-local.mjs  # 隨時重新執行
```

## Python — 從 wheel 或 editable 安裝

### Editable 安裝（最快的迭代方式）

從 `py/` 資料夾：

```bash
cd py
pip install -e .
```

程式碼編輯後無需重新安裝即可生效。

### Wheel 安裝（驗證發布的結構）

```bash
cd py
python -m build               # 產生 dist/imbrace-*.whl
pip install dist/imbrace-*.whl --force-reinstall
```

這能捕捉到 editable 安裝所隱藏的遺漏檔案、打包錯誤與匯入路徑問題。

### 執行測試

```bash
cd py
pip install -r tests/requirements.txt
python -m pytest tests/
```

對應[完整流程指南](/sdk/full-flow-guide/)的回歸測試：

```bash
cd test-pip-pkg/py
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m pytest tests/test_guide_flow.py -v
```

同一測試會以 `IMBRACE_API_KEY` 和 `IMBRACE_ACCESS_TOKEN` 兩種驗證模式執行。

## 疑難排解

**`Cannot find package '@imbrace/sdk'`** (TypeScript)
再次從測試資料夾內執行 `npm link @imbrace/sdk`。如果你刪除並重新安裝 `ts/` 中的 `node_modules`，連結可能會中斷。

**`ERR_MODULE_NOT_FOUND` 指向某個 dist 檔案** (TypeScript)
尚未執行建置，或在未重新建置的情況下新增了原始檔。在 `ts/` 中執行 `npm run build`。

**`ModuleNotFoundError: No module named 'imbrace'`** (Python)
套件未安裝在啟用的虛擬環境中。重新執行 `pip install -e .`（editable）或 `pip install dist/imbrace-*.whl`。

**即時呼叫出現 401 / 403**
你的憑證已過期、被撤銷或不正確。若是 API Key，請產生一組新的：

```bash
curl -X POST https://app-gateway.dev.imbrace.co/private/backend/v1/third_party_token \
  -H "x-access-token: <your_existing_token>" \
  -H "Content-Type: application/json" \
  -d '{"expirationDays": 30}'
```

其他執行時期錯誤請參閱[錯誤處理](/sdk/error-handling/)與[疑難排解](/guides/troubleshooting/)。
