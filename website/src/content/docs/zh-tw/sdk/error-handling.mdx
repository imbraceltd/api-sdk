SDK 拋出的所有錯誤都繼承自單一基底類型，因此你可以在一個地方捕獲任何 SDK 錯誤，或在需要不同反應時分支到特定的子類別。TypeScript 與 Python SDK 的階層結構完全相同。

## 錯誤階層

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```
Error
└── ImbraceError          （基底 — 捕獲所有 SDK 錯誤）
    ├── AuthError          （401, 403 — 憑證無效或過期）
    ├── ApiError           （4xx/5xx — 請求被伺服器拒絕）
    └── NetworkError       （超時、連線被拒、DNS 失敗）
```
</TabItem>
<TabItem label="Python">
```
Exception
└── ImbraceError          （基底 — 捕獲所有 SDK 錯誤）
    ├── AuthError          （401, 403 — 憑證無效或過期）
    ├── ApiError           （4xx/5xx — 請求被伺服器拒絕）
    └── NetworkError       （超時、連線被拒、DNS 失敗）
```
</TabItem>
</Tabs>

特定錯誤訊息與已知修復方式請參閱[疑難排解](/guides/troubleshooting/)。

## AuthError

當伺服器回傳 **401** 或 **403** 時拋出 — 憑證無效、過期或被撤銷。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import { AuthError } from "@imbrace/sdk";

try {
  const me = await client.platform.getMe();
} catch (e) {
  if (e instanceof AuthError) {
    console.error("請重新驗證：", e.message);
  }
}
```
</TabItem>
<TabItem label="Python">
```python
from imbrace import AuthError

try:
    me = client.platform.get_me()
except AuthError as e:
    print(f"驗證失敗：{e}")
    # 重新驗證後再重試
```
</TabItem>
</Tabs>

:::caution
`AuthError` **絕不會重試**。SDK 在收到 401/403 時會立即拋出 — 請先修正憑證再重試。關於憑證策略，請參閱[驗證](/sdk/authentication/)。
:::

## ApiError

所有其他 **4xx 與 5xx** 回應時拋出（429/5xx 的重試耗盡後）。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import { ApiError } from "@imbrace/sdk";

try {
  await client.marketplace.getProduct("nonexistent_id");
} catch (e) {
  if (e instanceof ApiError) {
    console.error(`HTTP ${e.statusCode}: ${e.message}`);
    // 例如 "HTTP 404: 找不到產品"
  }
}
```

| 屬性 | 型別 | 說明 |
| ------------ | -------- | ---------------------------------- |
| `statusCode` | `number` | HTTP 狀態碼 |
| `message` | `string` | 伺服器回應中的錯誤訊息 |
</TabItem>
<TabItem label="Python">
```python
from imbrace import ApiError

try:
    client.marketplace.get_product("nonexistent_id")
except ApiError as e:
    print(f"HTTP {e.status_code}: {e}")
    # 例如 "HTTP 404: 找不到產品"
```

| 屬性 | 型別 | 說明 |
| ------------- | ----- | ---------------------------------- |
| `status_code` | `int` | HTTP 狀態碼 |
| `message` | `str` | 伺服器回應中的錯誤訊息 |
</TabItem>
</Tabs>

## NetworkError

當請求從未到達伺服器時拋出 — 超時、DNS 失敗或連線重置。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import { NetworkError } from "@imbrace/sdk";

try {
  await client.platform.getMe();
} catch (e) {
  if (e instanceof NetworkError) {
    console.error("無法連線至 baseUrl：", e.message);
    // 例如 "請求在 30000ms 後超時"
  }
}
```
</TabItem>
<TabItem label="Python">
```python
from imbrace import NetworkError

try:
    client.platform.get_me()
except NetworkError as e:
    print(f"網路錯誤：{e}")
    # 例如 "請求在 30s 後超時"
```
</TabItem>
</Tabs>

## 捕獲所有 SDK 錯誤

匯入基底類型，在單一區塊中處理任何 SDK 引發的錯誤：

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import {
  ImbraceClient,
  ImbraceError,
  AuthError,
  ApiError,
  NetworkError,
} from "@imbrace/sdk";

try {
  await client.platform.getMe();
} catch (e) {
  if (e instanceof AuthError) return handleAuthError(e);
  if (e instanceof ApiError) return handleApiError(e);
  if (e instanceof NetworkError) return handleNetworkError(e);
  if (e instanceof ImbraceError) return handleUnknown(e);
  throw e; // 重新拋出非 SDK 錯誤
}
```
</TabItem>
<TabItem label="Python">
```python
from imbrace import ImbraceError, AuthError, ApiError, NetworkError

try:
    result = client.platform.get_me()
except AuthError as e:
    handle_auth_error(e)
except ApiError as e:
    handle_api_error(e)
except NetworkError as e:
    handle_network_error(e)
except ImbraceError as e:
    handle_unknown_sdk_error(e)
```
</TabItem>
</Tabs>

## 自動重試行為

兩個 SDK 的 HTTP 傳輸層都會以指數退避重試暫時性失敗。重試次數在語言之間略有不同，但條件完全相同。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
| 條件 | 行為 |
| --------------------------- | ---------------------------------------- |
| HTTP **429**（速率限制） | 最多重試 2 次 |
| HTTP **5xx**（伺服器錯誤） | 最多重試 2 次 |
| 網路錯誤 / 超時 | 最多重試 2 次 |
| HTTP **401 / 403** | 不重試 — 立即拋出 `AuthError` |
| HTTP **4xx**（其他） | 不重試 — 立即拋出 `ApiError` |

**退避：** `2^重試次數` 秒（2s → 4s）。最壞情況總計：3 次嘗試。
</TabItem>
<TabItem label="Python">
| 條件 | 行為 |
| --------------------------- | ---------------------------------------- |
| HTTP **429**（速率限制） | 最多重試 3 次 |
| HTTP **5xx**（伺服器錯誤） | 最多重試 3 次 |
| 網路錯誤 / 超時 | 最多重試 3 次 |
| HTTP **401 / 403** | 不重試 — 立即拋出 `AuthError` |
| HTTP **4xx**（其他） | 不重試 — 立即拋出 `ApiError` |

**退避：** `2^重試次數` 秒（2s → 4s → 8s）。最壞情況總計：4 次嘗試。
</TabItem>
</Tabs>

## 請求取消 (TypeScript)

傳遞 `AbortSignal` 以取消進行中的請求：

```typescript
const controller = new AbortController();
setTimeout(() => controller.abort(), 5000);

try {
  const result = await client.marketplace.listProducts({ page: 1 });
} catch (e) {
  if (e instanceof NetworkError && e.message.includes("aborted")) {
    console.log("請求已取消");
  }
}
```

## 非同步錯誤處理 (Python)

非同步客戶端會拋出相同的例外類型：

```python
from imbrace import AsyncImbraceClient, AuthError, ApiError

async with AsyncImbraceClient() as client:
    try:
        me = await client.platform.get_me()
    except AuthError:
        print("請重新驗證")
    except ApiError as e:
        print(f"[{e.status_code}] {e}")
```

## 最佳實踐

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
// 1. 始終單獨處理 AuthError — 需要重新整理憑證
// 2. 記錄 ApiError.statusCode — 400 = 參數錯誤，404 = 找不到，409 = 衝突
// 3. 將頂層入口點包裝在 try/catch 中
// 4. 不要對 AuthError 進行重試 — 在憑證修正前沒有幫助

async function safeGetMe(client: ImbraceClient) {
  try {
    return await client.platform.getMe();
  } catch (e) {
    if (e instanceof AuthError) {
      await refreshCredentials();
      return await client.platform.getMe();
    }
    throw e;
  }
}
```
</TabItem>
<TabItem label="Python">
```python
# 1. 使用情境管理器以確保連線確定性地關閉
with ImbraceClient() as client:
    ...

# 2. 單獨處理 AuthError — 在重試前需要重新整理憑證
def safe_get_me(client):
    try:
        return client.platform.get_me()
    except AuthError:
        refresh_credentials(client)
        return client.platform.get_me()

# 3. 根據 status_code 分支處理 ApiError
try:
    client.marketplace.create_order(data)
except ApiError as e:
    if e.status_code == 409:
        print("訂單已存在")
    elif e.status_code == 400:
        print(f"資料無效：{e}")
```
</TabItem>
</Tabs>
