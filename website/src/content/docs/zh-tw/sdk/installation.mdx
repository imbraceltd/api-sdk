### 安裝

<VersionOnly version="1.0.2">
<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```bash
    npm install @imbrace/sdk@1.0.2
    # 或
    yarn add @imbrace/sdk@1.0.2
    # 或
    pnpm add @imbrace/sdk@1.0.2
    ```

    需要 Node.js 18+（或任何支援原生 `fetch` 與 `ReadableStream` 的瀏覽器）。
  </TabItem>
  <TabItem label="Python">
    ```bash
    pip install imbrace==1.0.2
    # 或
    uv add imbrace==1.0.2
    ```

    需要 Python 3.9+。
  </TabItem>
</Tabs>
</VersionOnly>

### 初始化客戶端

<AuthOnly mode="access-token">
<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```typescript
    import { ImbraceClient } from "@imbrace/sdk"

    // 建立在 Imbrace 上 — Imbrace 就是你的後端，終端使用者透過 OTP 登入
    const client = new ImbraceClient({
      accessToken: "acc_your_token",
      baseUrl: "https://app-gatewayv2.imbrace.co",
    })
    ```

    客戶端是有狀態的 — 建立一次並在整個應用程式中重複使用。
  </TabItem>
  <TabItem label="Python">
    ```python
    from imbrace import ImbraceClient

    # 建立在 Imbrace 上 — Imbrace 就是你的後端，終端使用者透過 OTP 登入
    with ImbraceClient(access_token="acc_your_token") as client:
        ...
    ```

    Python 也匯出 `AsyncImbraceClient`（使用 `async with ...`）供 FastAPI 等非同步架構使用。

    情境管理器會自動關閉底層的 HTTP 連線池。
  </TabItem>
</Tabs>
</AuthOnly>

### 環境變數

SDK **不會**自動讀取環境變數。請將憑證直接傳遞給建構函式，如果你將憑證存放在 `.env` 中，請使用載入器（`dotenv` / 你框架的環境變數處理方式）。

| 變數 | 用途 |
|---|---|
| `IMBRACE_API_KEY` | 你的 API Key（組織層級憑證） |
| `IMBRACE_ACCESS_TOKEN` | 使用者的存取權杖（每個工作階段的憑證） |
| `IMBRACE_BASE_URL` | 覆寫 Gateway URL（預設：`https://app-gatewayv2.imbrace.co`） |

組織 ID 已編碼於 API Key 和 Access Token 內部 — 你無需將 `organizationId`/`organization_id` 傳遞給 SDK。詳細說明請參閱[驗證](/sdk/authentication/)。
