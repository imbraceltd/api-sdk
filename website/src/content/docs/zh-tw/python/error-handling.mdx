所有 SDK 例外都繼承自 `ImbraceError`，讓您可以在單一的 `except` 子句中捕獲所有 SDK 錯誤。

### 例外階層

```
Exception
└── ImbraceError          (base — catch-all SDK errors)
    ├── AuthError          (401, 403 — 認證無效或過期)
    ├── ApiError           (4xx/5xx — 伺服器拒絕請求)
    └── NetworkError       (timeout、連線被拒、DNS 失敗)
```

---

### AuthError

當伺服器返回 **401** 或 **403** 時引發。

```python
from imbrace import ImbraceClient, AuthError

try:
    me = client.platform.get_me()
except AuthError as e:
    # 認證無效或已過期
    print(f"認證失敗: {e}")
    # 需要重新認證後再試
```

:::caution
`AuthError` **永遠不會重試**。SDK 在收到 401/403 時會立即引發 — 需要先修復認證資料再重試。
:::

---

### ApiError

對所有其他 **4xx 和 5xx** 錯誤（429/5xx 重試後）引發。

```python
from imbrace import ApiError

try:
    client.marketplace.get_product("id_khong_ton_tai")
except ApiError as e:
    print(f"HTTP {e.status_code}: {e}")
    # 例如："HTTP 404: Product not found"
    ```

| Attribute     | Type  | 說明                            |
| ------------- | ----- | -------------------------------- |
| `status_code` | `int` | HTTP 狀態碼                 |
| `message`     | `str` | 伺服器回應的錯誤訊息 |

---

### NetworkError

當請求無法到達伺服器時引發 — timeout、DNS 失敗或連線重置。

```python
from imbrace import NetworkError

try:
    client.platform.get_me()
except NetworkError as e:
    print(f"網路錯誤: {e}")
    # 例如："Request timed out after 30s"

### 捕獲所有 SDK 錯誤

```python
from imbrace import ImbraceError, AuthError, ApiError, NetworkError

try:
    result = client.platform.get_me()
except AuthError as e:
    handle_auth_error(e)
except ApiError as e:
    handle_api_error(e)
except NetworkError as e:
    handle_network_error(e)
except ImbraceError as e:
    handle_unknown_error(e)
```

### 自動重試行為

| 條件                   | 動作                                    |
| --------------------------- | -------------------------------------------- |
| HTTP **429** (rate limit)   | 最多重試 3 次                           |
| HTTP **5xx** (server error) | 最多重試 3 次                           |
| 網路錯誤 / timeout         | 最多重試 3 次                           |
| HTTP **401 / 403**          | 不重試 — 立即引發 `AuthError` |
| HTTP **4xx** (其他)        | 不重試 — 立即引發 `ApiError`  |

**退避公式：** `2^retryCount` 秒（2s → 4s → 8s）。

最壞情況：4 次嘗試（1 次初始 + 3 次重試）後才引發。

:::note
Python SDK 最多重試 **3 次**（總共 4 次），而 TypeScript SDK 最多重試 **2 次**（總共 3 次）。兩者都使用相同的指數退避公式。
:::

### Async 錯誤處理

例外狀況完全相同 — 只需加上 `await`：

```python
from imbrace import AsyncImbraceClient, AuthError, ApiError

async with AsyncImbraceClient() as client:
    try:
        me = await client.platform.get_me()
    except AuthError:
        print("需要重新認證")
    except ApiError as e:
        print(f"[{e.status_code}] {e}")
```

### 最佳實踐

```python
# 1. 使用 context manager 確保連線關閉
with ImbraceClient() as client:
    ...

# 2. 分開處理 AuthError — 認證資料需要 refresh 後再重試
def safe_get_me(client):
    try:
        return client.platform.get_me()
    except AuthError:
        refresh_credentials(client)
        return client.platform.get_me()  # refresh 後重試一次

# 3. 記錄 ApiError.status_code 以便 debug
try:
    client.marketplace.create_product(data)
except ApiError as e:
    if e.status_code == 409:
        print("產品已存在")
    elif e.status_code == 400:
        print(f"資料無效: {e}")

# 4. 不要捕獲 ImbraceError 並靜默忽略 — 永遠記錄或重新引發
```
