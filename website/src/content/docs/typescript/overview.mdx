title: TypeScript SDK — Overview

---

The `@imbrace/sdk` package is the official TypeScript/JavaScript client for the Imbrace Gateway. It provides fully-typed access to all Imbrace services for both server-side (Node.js) and client-side (browser) contexts.

### Key Features

| Feature                  | Detail                                                                               |
| ------------------------ | ------------------------------------------------------------------------------------ |
| **Full Type Safety**     | 100% TypeScript — every request and response is typed via shared `types/` interfaces |
| **Dual Authentication**  | API Key or Access Token — see [Authentication](/typescript/authentication/) for which to pick |
| **Auto Retry**           | 429 and 5xx errors retry up to 2 times with exponential backoff                      |
| **Request Cancellation** | AbortSignal propagation for in-flight request cancellation                           |
| **Streaming AI**         | AsyncGenerator-based streaming for AI completions and SSE chat                       |

### Available Resources

| Namespace | Purpose |
|---|---|
| `client.aiAgent` | Streaming AI chat, embeddings, parquet data, chat session management |
| `client.chatAi` | LLM completions, streaming, models, knowledge, files, folders |
| `client.activepieces` | Workflow automation — flows, triggers, runs, connections |
| `client.boards` | CRM boards — CRUD, items, fields, search, segments, CSV export |
| `client.contacts` | Contact management |
| `client.conversations` | Conversation views, assignment, status |
| `client.messages` | Send and list messages |
| `client.channel` | Channel listing and messaging |
| `client.platform` | Users, organizations, permissions |
| `client.ai` | Raw OpenAI-compatible completions and embeddings |

### When to Use

| Scenario                                   | Recommended Auth                            |
| ------------------------------------------ | ------------------------------------------- |
| [Wrap Imbrace](/typescript/authentication/#wrap-imbrace--use-api-key) — Imbrace is a feature inside your own backend | `apiKey` → `x-api-key` header               |
| [Build on Imbrace](/typescript/authentication/#build-on-imbrace--use-access-token) — Imbrace IS your backend (your users log in via Imbrace) | `accessToken` → `x-access-token` header |

See [Authentication](/typescript/authentication/) for the full decision tree and examples.
