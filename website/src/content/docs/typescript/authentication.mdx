---
title: TypeScript SDK — Authentication
description: API Key and Access Token authentication for the Imbrace TypeScript SDK.
---

The SDK supports two authentication strategies. Pass either `apiKey` or `accessToken` when constructing `ImbraceClient` — the transport handles headers automatically.

### Authentication Strategies

| Strategy         | When to use                              | Header sent              |
| ---------------- | ---------------------------------------- | ------------------------ |
| **API Key**      | Backend scripts, cron jobs, server calls | `x-api-key: sk-xxx...`   |
| **Access Token** | After user login (OTP or password)       | `x-access-token: acc_...`|

---

### API Key (Server-Side)

```typescript
import { ImbraceClient } from "@imbrace/sdk";

const client = new ImbraceClient({
  apiKey: process.env.IMBRACE_API_KEY,
});
```

---

### Access Token (Client-Side)

Use after a user authenticates:

```typescript
const client = new ImbraceClient({
  accessToken: "acc_xxxxxxxxxxxxx",
});
```

---

### OTP Login Flow

The recommended approach for user-facing apps — the SDK stores the token automatically after login.

```typescript
import { ImbraceClient } from "@imbrace/sdk";

const client = new ImbraceClient();

// Step 1: Trigger OTP delivery
await client.requestOtp("user@example.com");

// Step 2: User enters OTP — token stored automatically
await client.loginWithOtp("user@example.com", "123456");

// Step 3: All subsequent calls are authenticated
const me = await client.platform.getMe();
```

Email + password login is also supported:

```typescript
await client.login("user@example.com", "password123");
```

---

### Token Management

```typescript
// Replace token (e.g., after OAuth token refresh)
client.setAccessToken("new_token...");

// Clear token (e.g., on logout)
client.clearAccessToken();
```

---

### Configuration Priority

The client merges configuration in this order (highest to lowest priority):

1. **Explicit constructor options** — `new ImbraceClient({ apiKey: "..." })`
2. **Environment variables** — `IMBRACE_API_KEY`, `IMBRACE_GATEWAY_URL`
3. **Defaults** — `env: "stable"`, gateway `https://app-gatewayv2.imbrace.co`
