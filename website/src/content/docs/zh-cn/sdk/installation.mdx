### 安装

<VersionOnly version="1.0.2">
<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```bash
    npm install @imbrace/sdk@1.0.2
    # 或者
    yarn add @imbrace/sdk@1.0.2
    # 或者
    pnpm add @imbrace/sdk@1.0.2
    ```

    需要 Node.js 18+（或任何支持原生 `fetch` 和 `ReadableStream` 的浏览器）。
  </TabItem>
  <TabItem label="Python">
    ```bash
    pip install imbrace==1.0.2
    # 或者
    uv add imbrace==1.0.2
    ```

    需要 Python 3.9+。
  </TabItem>
</Tabs>
</VersionOnly>

### 初始化客户端

<AuthOnly mode="access-token">
<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```typescript
    import { ImbraceClient } from "@imbrace/sdk"

    // 基于 Imbrace 构建——Imbrace 就是你的后端，终端用户通过 OTP 登录
    const client = new ImbraceClient({
      accessToken: "acc_your_token",
      baseUrl: "https://app-gatewayv2.imbrace.co",
    })
    ```

    客户端是有状态的——创建一次并在整个应用中复用。
  </TabItem>
  <TabItem label="Python">
    ```python
    from imbrace import ImbraceClient

    # 基于 Imbrace 构建——Imbrace 就是你的后端，终端用户通过 OTP 登录
    with ImbraceClient(access_token="acc_your_token") as client:
        ...
    ```

    Python 还导出了 `AsyncImbraceClient`（`async with ...`），适用于 FastAPI 等异步框架。

    上下文管理器会自动关闭底层的 HTTP 连接池。
  </TabItem>
</Tabs>
</AuthOnly>

### 环境变量

SDK **不会**自动读取环境变量。请直接将凭证传递给构造函数，并使用加载器（`dotenv`/你框架的环境变量处理方式）来管理 `.env` 中的变量。

| 变量 | 用途 |
|---|---|
| `IMBRACE_API_KEY` | 你的 API Key（组织级凭证） |
| `IMBRACE_ACCESS_TOKEN` | 用户的访问令牌（按会话的凭证） |
| `IMBRACE_BASE_URL` | 覆盖网关 URL（默认：`https://app-gatewayv2.imbrace.co`） |

组织 ID 已编码在 API Key 和访问令牌中——你无需向 SDK 传递 `organizationId`/`organization_id`。详情参见[认证](/sdk/authentication/)。
