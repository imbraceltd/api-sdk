SDK 抛出的所有错误都继承自一个单一基类型，因此你可以在一个地方捕获任何 SDK 错误，或在需要不同响应时分支到特定子类。TypeScript 和 Python SDK 的错误层次结构完全相同。

## 错误层次结构

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```
Error
└── ImbraceError          (基类 — 捕获所有 SDK 错误)
    ├── AuthError          (401, 403 — 凭证无效或已过期)
    ├── ApiError           (4xx/5xx — 请求被服务器拒绝)
    └── NetworkError       (超时、连接被拒绝、DNS 故障)
```
</TabItem>
<TabItem label="Python">
```
Exception
└── ImbraceError          (基类 — 捕获所有 SDK 错误)
    ├── AuthError          (401, 403 — 凭证无效或已过期)
    ├── ApiError           (4xx/5xx — 请求被服务器拒绝)
    └── NetworkError       (超时、连接被拒绝、DNS 故障)
```
</TabItem>
</Tabs>

有关具体错误信息和已知修复，参见[故障排除](/guides/troubleshooting/)。

## AuthError

当服务器返回 **401** 或 **403** 时抛出——凭证无效、已过期或被撤销。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import { AuthError } from "@imbrace/sdk";

try {
  const me = await client.platform.getMe();
} catch (e) {
  if (e instanceof AuthError) {
    console.error("请重新认证：", e.message);
  }
}
```
</TabItem>
<TabItem label="Python">
```python
from imbrace import AuthError

try:
    me = client.platform.get_me()
except AuthError as e:
    print(f"认证失败：{e}")
    # 重新认证后重试
```
</TabItem>
</Tabs>

:::caution
`AuthError` **从不重试**。SDK 在 401/403 时立即抛出/引发——在重试之前修复凭证。有关凭证策略，参见[认证](/sdk/authentication/)。
:::

## ApiError

在所有其他 **4xx 和 5xx** 响应时抛出（针对 429/5xx 的重试用尽后）。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import { ApiError } from "@imbrace/sdk";

try {
  await client.marketplace.getProduct("nonexistent_id");
} catch (e) {
  if (e instanceof ApiError) {
    console.error(`HTTP ${e.statusCode}: ${e.message}`);
    // 例如 "HTTP 404: Product not found"
  }
}
```

| 属性         | 类型     | 说明                        |
| ------------ | -------- | --------------------------- |
| `statusCode` | `number` | HTTP 状态码                 |
| `message`    | `string` | 来自服务器响应的错误消息    |
</TabItem>
<TabItem label="Python">
```python
from imbrace import ApiError

try:
    client.marketplace.get_product("nonexistent_id")
except ApiError as e:
    print(f"HTTP {e.status_code}: {e}")
    # 例如 "HTTP 404: Product not found"
```

| 属性          | 类型  | 说明                        |
| ------------- | ----- | --------------------------- |
| `status_code` | `int` | HTTP 状态码                 |
| `message`    | `str` | 来自服务器响应的错误消息    |
</TabItem>
</Tabs>

## NetworkError

当请求从未到达服务器时抛出——超时、DNS 故障或连接重置。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import { NetworkError } from "@imbrace/sdk";

try {
  await client.platform.getMe();
} catch (e) {
  if (e instanceof NetworkError) {
    console.error("无法连接至 baseUrl：", e.message);
    // 例如 "Request timed out after 30000ms"
  }
}
```
</TabItem>
<TabItem label="Python">
```python
from imbrace import NetworkError

try:
    client.platform.get_me()
except NetworkError as e:
    print(f"网络错误：{e}")
    # 例如 "Request timed out after 30s"
```
</TabItem>
</Tabs>

## 捕获所有 SDK 错误

导入基类型以在单个代码块中处理所有 SDK 产生的错误：

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
import {
  ImbraceClient,
  ImbraceError,
  AuthError,
  ApiError,
  NetworkError,
} from "@imbrace/sdk";

try {
  await client.platform.getMe();
} catch (e) {
  if (e instanceof AuthError) return handleAuthError(e);
  if (e instanceof ApiError) return handleApiError(e);
  if (e instanceof NetworkError) return handleNetworkError(e);
  if (e instanceof ImbraceError) return handleUnknown(e);
  throw e; // 重新抛出非 SDK 错误
}
```
</TabItem>
<TabItem label="Python">
```python
from imbrace import ImbraceError, AuthError, ApiError, NetworkError

try:
    result = client.platform.get_me()
except AuthError as e:
    handle_auth_error(e)
except ApiError as e:
    handle_api_error(e)
except NetworkError as e:
    handle_network_error(e)
except ImbraceError as e:
    handle_unknown_sdk_error(e)
```
</TabItem>
</Tabs>

## 自动重试行为

两个 SDK 的 HTTP 传输层都会以指数退避方式重试临时故障。重试次数在语言间略有不同，但触发条件相同。

<Tabs syncKey="lang">
<TabItem label="TypeScript">
| 条件                           | 操作                                   |
| ------------------------------ | -------------------------------------- |
| HTTP **429**（限流）           | 最多重试 2 次                          |
| HTTP **5xx**（服务器错误）     | 最多重试 2 次                          |
| 网络错误 / 超时                | 最多重试 2 次                          |
| HTTP **401 / 403**             | 不重试 — 立即抛出 `AuthError`          |
| HTTP **4xx**（其他）           | 不重试 — 立即抛出 `ApiError`           |

**退避：** 每次尝试之间等待 `2^重试次数` 秒（2 秒 → 4 秒）。最坏情况总计：3 次尝试。
</TabItem>
<TabItem label="Python">
| 条件                           | 操作                                   |
| ------------------------------ | -------------------------------------- |
| HTTP **429**（限流）           | 最多重试 3 次                          |
| HTTP **5xx**（服务器错误）     | 最多重试 3 次                          |
| 网络错误 / 超时                | 最多重试 3 次                          |
| HTTP **401 / 403**             | 不重试 — 立即抛出 `AuthError`          |
| HTTP **4xx**（其他）           | 不重试 — 立即抛出 `ApiError`           |

**退避：** 每次尝试之间等待 `2^重试次数` 秒（2 秒 → 4 秒 → 8 秒）。最坏情况总计：4 次尝试。
</TabItem>
</Tabs>

## 请求取消（TypeScript）

传入 `AbortSignal` 以取消正在进行的请求：

```typescript
const controller = new AbortController();
setTimeout(() => controller.abort(), 5000);

try {
  const result = await client.marketplace.listProducts({ page: 1 });
} catch (e) {
  if (e instanceof NetworkError && e.message.includes("aborted")) {
    console.log("请求已取消");
  }
}
```

## 异步错误处理（Python）

异步客户端抛出相同的异常类型：

```python
from imbrace import AsyncImbraceClient, AuthError, ApiError

async with AsyncImbraceClient() as client:
    try:
        me = await client.platform.get_me()
    except AuthError:
        print("请重新认证")
    except ApiError as e:
        print(f"[{e.status_code}] {e}")
```

## 最佳实践

<Tabs syncKey="lang">
<TabItem label="TypeScript">
```typescript
// 1. 始终单独处理 AuthError —— 凭证需要刷新
// 2. 记录 ApiError.statusCode —— 400 = 参数错误，404 = 未找到，409 = 冲突
// 3. 将顶层入口点包裹在 try/catch 中
// 4. 不要在 AuthError 上重试 —— 在修复凭证之前无济于事

async function safeGetMe(client: ImbraceClient) {
  try {
    return await client.platform.getMe();
  } catch (e) {
    if (e instanceof AuthError) {
      await refreshCredentials();
      return await client.platform.getMe();
    }
    throw e;
  }
}
```
</TabItem>
<TabItem label="Python">
```python
# 1. 使用上下文管理器以确保连接确定性关闭
with ImbraceClient() as client:
    ...

# 2. 单独处理 AuthError —— 重试前需要刷新凭证
def safe_get_me(client):
    try:
        return client.platform.get_me()
    except AuthError:
        refresh_credentials(client)
        return client.platform.get_me()

# 3. 根据 ApiError 的 status_code 进行分支处理
try:
    client.marketplace.create_order(data)
except ApiError as e:
    if e.status_code == 409:
        print("订单已存在")
    elif e.status_code == 400:
        print(f"数据无效：{e}")
```
