SDK 支持两种凭证类型。将 `apiKey`/`api_key` 或 `accessToken`/`access_token` 传递给客户端构造函数——传输层会自动处理请求头。

### 我应该使用哪种凭证？

选择取决于 **Imbrace 在你的产品中扮演什么角色**：

#### 基于 Imbrace 构建 → 使用 Access Token

Imbrace 就是你的后端。你的终端用户登录 Imbrace（通过 OTP 或密码）；他们的 `acc_...` 访问令牌是 SDK 在每个请求中使用的凭证。Imbrace 的认证、数据库和业务逻辑（助手、知识库、工作流、AI 代理）都代表实际登录用户运行。

典型产品：聊天组件、仪表盘或移动应用，其中每个用户的身份就是一个 Imbrace 用户。Imbrace 记录每个用户的历史记录、权限和审计日志。

#### 封装 Imbrace → 使用 API Key

你有自己的后端、自己的用户、自己的数据库。Imbrace 是你的服务调用的一个能力——"使用 Imbrace 的 AI 来回答这个问题"——与你所做的其他事情并行。你的终端用户从不直接接触 Imbrace；你的服务使用一个组织级的 `api_...` 密钥（由 Imbrace 组织管理员签发）来调用 Imbrace。Imbrace 只看到一个身份（API Key 对应的服务用户），而每个用户的归属由你自行处理。

典型产品：现有的 CRM、工单系统或内部工具，将 Imbrace 作为功能嵌入。一个凭证保存在你的环境变量文件中，服务于每个请求。

| | 基于 Imbrace 构建（Access Token） | 封装 Imbrace（API Key） |
| --- | --- | --- |
| **谁负责认证？** | Imbrace（OTP/密码登录） | 你自己（你自己的认证） |
| **用户是谁？** | Imbrace 用户 | 你的用户 |
| **谁的数据库是权威？** | Imbrace 的 | 你的 |
| **Imbrace 看到的身份** | 实际的终端用户 | 一个服务账号 |
| **上游的按用户归属** | 内置 | 由你决定 |
| **凭证生命周期** | 会话级别；按需刷新 | 长期有效；不会自动过期 |
| **发送的请求头** | `x-access-token: acc_...` | `x-api-key: api_...` |

> 两种凭证类型都是一等公民。大多数资源接受任一类型。少数依赖于用户上下文的特性（文档产物、按用户的聊天历史）仅在 access token 下有意义。

### 请求头参考

| 凭证 | 发送的请求头 |
| --- | --- |
| `apiKey` / `api_key` | `x-api-key: api_xxx...` |
| `accessToken` / `access_token` | `x-access-token: acc_xxx...` |

组织上下文**编码在凭证本身中**——每个 API Key 和每个访问令牌都绑定到唯一一个组织。网关会根据你发送的凭证在每个请求上解析组织身份。你无需向 SDK 传递 `organizationId`/`organization_id`。

关于凭证设置的分步指南（环境变量、dotenv、密钥），参见[设置指南](/getting-started/setup/#configure-credentials)。

### OTP 登录流程

使用此流程通过邮箱 OTP 认证用户并获取会话令牌。SDK 在登录后会自动存储令牌。

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```typescript
    import { ImbraceClient } from "@imbrace/sdk"

    const client = new ImbraceClient({
      baseUrl: "https://app-gatewayv2.imbrace.co",
    })

    // 步骤 1：向用户邮箱发送 OTP
    await client.requestOtp("user@example.com")

    // 步骤 2：用户提交收到的 OTP
    const loginRes = await client.loginWithOtp("user@example.com", "ABC123")
    // loginRes 包含一个短期有效的 login_acc_... 令牌（已在内部存储）

    // 步骤 3：兑换为长期有效的访问令牌
    const { token, refresh_token } = await client.auth.exchangeAccessToken("org_your_org_id")

    // 步骤 4：激活令牌用于后续所有调用
    client.setAccessToken(token)

    // 现在可以使用任何资源
    const { data: boards } = await client.boards.list()
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    from imbrace import ImbraceClient

    client = ImbraceClient()

    # 步骤 1：向用户邮箱发送 OTP
    client.request_otp("user@example.com")

    # 步骤 2：用户输入 OTP——令牌自动存储
    client.login_with_otp("user@example.com", "123456")

    # 步骤 3：后续所有调用均已认证
    me = client.platform.get_me()
    ```

    异步变体：

    ```python
    from imbrace import AsyncImbraceClient

    async with AsyncImbraceClient() as client:
        await client.request_otp("user@example.com")
        await client.login_with_otp("user@example.com", "123456")
        me = await client.platform.get_me()
    ```
  </TabItem>
</Tabs>

### 上下文管理器（仅 Python）

始终将 Python 客户端包装在上下文管理器中，以便底层的 `httpx` 连接池能够干净地关闭：

```python
# 同步
with ImbraceClient(api_key="api_xxx") as client:
    me = client.platform.get_me()

# 异步
async with AsyncImbraceClient(api_key="api_xxx") as client:
    me = await client.platform.get_me()
```
