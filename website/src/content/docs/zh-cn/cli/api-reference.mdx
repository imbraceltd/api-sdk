The local API server (Hono, Bun, port `3456`) acts as a proxy to the Imbrace platform (`https://app-gatewayv2.imbrace.co`).

**Base URL:** `http://localhost:3456`

## Authentication

All endpoints under `/data-board/*`, `/ai-agent/*`, and `/workflow/*` require:

**Header:** `Authorization: Bearer <credential>`

Where `credential` is either an API key (`sk-` or `api_`) or a JWT token (from email+password login). The middleware verifies every request by calling `client.boards.list()`.

<Aside type="tip">
  Public endpoints: `GET /` and `POST /auth/login`.
</Aside>

## Auth

### `POST /auth/login`

**Body (choose one):**
```json
{ "apiKey": "api_xxx..." }
{ "email": "user@example.com", "password": "mypass" }
```

**Response (200):**
```json
{ "ok": true, "method": "api-key", "message": "Authenticated with API key", "credential": "api_xxx..." }
{ "ok": true, "method": "password", "message": "Logged in as ...", "credential": "<jwt>", "email": "..." }
```

**Errors:** `400` (missing fields), `401` (invalid credentials).

## Data Board

All endpoints require auth.

| Method | Path | Description |
|---|---|---|
| `GET` | `/data-board/list` | List all boards |
| `GET` | `/data-board/:id` | Get board by ID |
| `POST` | `/data-board/create` | Create board `{ "name": "..." }` |
| `GET` | `/data-board/:id/fields` | List fields of a board |
| `POST` | `/data-board/:id/fields` | Create field `{ "name": "...", "type": "..." }` |
| `GET` | `/data-board/:id/items?limit=&skip=&q=` | List/search items |
| `POST` | `/data-board/:id/items` | Create item `{ "fields": [...] }` |
| `PUT` | `/data-board/:id/items/:itemId` | Update item `{ "data": [...] }` |
| `DELETE` | `/data-board/:id/items/:itemId` | Delete item |
| `GET` | `/data-board/:id/search?q=&limit=` | Search items (Meilisearch) |
| `GET` | `/data-board/:id/export-csv` | Export board as CSV (text) |

## Error Responses

```json
{ "ok": false, "message": "<error description>" }
```

| Status | Meaning |
|---|---|
| `400` | Bad request |
| `401` | Authentication failed |
| `404` | Route or resource not found |
| `500` | Internal server error |
