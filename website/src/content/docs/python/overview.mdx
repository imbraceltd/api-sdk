The `imbrace` package is the official Python client for the Imbrace Gateway. It provides both **synchronous** and **asynchronous** clients, making it suitable for traditional Django/Flask apps and modern async frameworks like FastAPI.

### Key Features

| Feature                 | Detail                                                                                       |
| ----------------------- | -------------------------------------------------------------------------------------------- |
| **Dual Client**         | `ImbraceClient` (sync, `httpx.Client`) and `AsyncImbraceClient` (async, `httpx.AsyncClient`) |
| **Full Typing**         | All requests and responses typed via Pydantic models and type hints                          |
| **Dual Authentication** | API Key (server-to-server) or Access Token (JWT / legacy token)                              |
| **Auto Retry**          | 429 and 5xx errors retry up to 3 times with exponential backoff                              |
| **Pagination Helper**   | `iterate_paged()` yields items one by one across all pages                                   |
| **Context Manager**     | `with`/`async with` syntax for automatic connection cleanup                                  |
| **Multi-Environment**   | `develop` / `sandbox` / `stable` presets with per-service URL overrides                      |

### How It Works

```
Your Code
   │
   ▼
ImbraceClient / AsyncImbraceClient     ← single entry point
   ├── TokenManager                    ← thread-safe token storage
   └── HttpTransport / AsyncHttpTransport
         ├── Auth strategy selection   ← API Key / JWT Bearer / x-access-token
         ├── Retry with backoff        ← 429 and 5xx
         └── Connection pool (httpx)   ← keepalive, max_connections=10
               │
               ▼
         Imbrace Gateway               ← app-gatewayv2.imbrace.co
```

### Internal Architecture

| Layer            | File                           | Responsibility                                     |
| ---------------- | ------------------------------ | -------------------------------------------------- |
| Entry Points     | `client.py`, `async_client.py` | Instantiates resources, login helpers              |
| HTTP Transport   | `http.py`                      | Auth headers, retry, timeout, error mapping        |
| Token Manager    | `auth/token_manager.py`        | Thread-safe token read/write                       |
| Service Registry | `service_registry.py`          | Resolves per-service URLs from env presets         |
| Resources        | `resources/*.py`               | Domain-specific API method wrappers (31 resources) |
| Types            | `types/*.py`                   | Pydantic models and dataclasses                    |
| Exceptions       | `exceptions.py`                | `AuthError`, `ApiError`, `NetworkError`            |
| Environments     | `environments.py`              | URL presets for each environment                   |

### Sync vs Async — When to Use Which

| Client               | Framework                      | Import                                   |
| -------------------- | ------------------------------ | ---------------------------------------- |
| `ImbraceClient`      | Django, Flask, scripts, Celery | `from imbrace import ImbraceClient`      |
| `AsyncImbraceClient` | FastAPI, asyncio, aiohttp      | `from imbrace import AsyncImbraceClient` |

:::tip
Both clients share identical resource interfaces — the only difference is `await` on method calls. You can swap between them without changing business logic.
:::
