### Install

```bash
pip install imbrace
```

Development mode (from source, includes dev tools):

```bash
cd py
pip install -e ".[dev]"
# Installs: pytest, ruff, mypy, pytest-httpx, python-dotenv
```

### Environment Setup

Copy the example env file and fill in your credentials:

```bash
cp .env.example .env
```

Open `.env` and set:

```env
IMBRACE_API_KEY=your-api-key-here
IMBRACE_BASE_URL=https://app-gatewayv2.imbrace.co
```

#### Get an API Key

Call the Imbrace auth endpoint with an existing access token:

```bash
curl -X POST https://app-gatewayv2.imbrace.co/private/backend/v1/third_party_token \
  -H "x-access-token: <your_existing_token>" \
  -H "Content-Type: application/json" \
  -d '{"expirationDays": 30}'
```

Response:

```json
{ "apiKey": { "apiKey": "sk-xxx..." } }
```

The value you need is `response.apiKey.apiKey`.

### Verify Installation

```python
from imbrace import ImbraceClient
client = ImbraceClient()
print("SDK ready")
```

### Environment Variables Reference

| Variable           | Required | Default  | Description                                              |
| ------------------ | -------- | -------- | -------------------------------------------------------- |
| `IMBRACE_API_KEY`  | Yes\*    | —        | Server-side API Key                                      |
| `IMBRACE_ENV`      | No       | `stable` | Preset: `develop`, `sandbox`, `stable`                   |
| `IMBRACE_BASE_URL` | No       | auto     | Override gateway URL (takes priority over `IMBRACE_ENV`) |
| `IMBRACE_ORG_ID`   | No       | —        | Organization ID (for integration tests)                  |

\*Or pass `access_token` directly for client-side usage.
