Hướng dẫn này cho phép test **package đã build** (`dist/`) chính xác như cách consumer cài từ npm — không phải raw source.

### Yêu Cầu

- Node 18+
- AWS credentials đã cấu hình (profile `imbrace`) nếu cần giải mã env vars từ Ansible

### Thiết Lập Ban Đầu (Một Lần)

#### 1. Build SDK

```bash
cd ts
npm install
npm run build
```

#### 2. Link package globally

```bash
# vẫn trong ts/
npm link
```

Lệnh này làm cho `@imbrace/sdk` available như thể được install từ npm.

#### 3. Link vào thư mục test

```bash
cd ts/tests/local
npm link @imbrace/sdk
```

#### 4. Cấu hình credentials

```bash
cp .env.example .env
```

Sau đó điền vào `.env`. Với môi trường dev, cách nhanh nhất là pull vars từ Ansible:

```bash
# từ ts/ root (hoặc nơi chứa secrets.enc.env)
AWS_PROFILE=imbrace sops -d ansible/dev/secrets.enc.env | grep IMBRACE >> tests/local/.env
```

Tối thiểu cần cho live calls:

| Biến                      | Lấy ở đâu                                                                                  |
| ------------------------- | ------------------------------------------------------------------------------------------ |
| `IMBRACE_API_KEY`         | Imbrace Portal, hoặc `POST /private/backend/v1/third_party_token` với access token hiện có |
| `IMBRACE_ORGANIZATION_ID` | UUID của org từ portal                                                                     |
| `IMBRACE_GATEWAY_URL`     | Để trống để dùng `app-gateway.dev.imbrace.co`                                              |

### Chạy Tests

```bash
cd ts/tests/local
node test-local.mjs
```

**Không có credentials** — chỉ chạy instantiation + resource surface checks (không có network).

**Với `IMBRACE_API_KEY` được set** — chạy toàn bộ live API checks tới gateway trong `IMBRACE_GATEWAY_URL`.

### Chuyển Đổi Môi Trường

Sửa `.env` (hoặc export inline):

| Môi trường | `IMBRACE_GATEWAY_URL`                    |
| ---------- | ---------------------------------------- |
| develop    | `https://app-gateway.dev.imbrace.co`     |
| sandbox    | `https://app-gateway.sandbox.imbrace.co` |
| stable     | `https://app-gatewayv2.imbrace.co`       |

```bash
IMBRACE_GATEWAY_URL=https://app-gateway.sandbox.imbrace.co node test-local.mjs
```

### Lặp Với SDK Changes

Mỗi khi sửa source files, rebuild và link sẽ tự dùng dist mới:

```bash
# ts/
npm run build

# tests/local
node test-local.mjs
```

Hoặc giữ compiler ở watch mode trong terminal riêng:

```bash
# terminal 1 — ts/
npm run dev

# terminal 2 — ts/tests/local
node test-local.mjs   # chạy lại bất cứ khi nào muốn
```

### Xử Lý Sự Cố

**`Cannot find package '@imbrace/sdk'`**

Chạy `npm link @imbrace/sdk` từ bên trong `tests/local/` lại. Link có thể bị break nếu bạn xóa và cài lại `node_modules` trong `ts/`.

**`ERR_MODULE_NOT_FOUND` cho dist file**

Build chưa chạy, hoặc source file mới được thêm mà chưa rebuild. Chạy `npm run build` trong `ts/`.

**401 / 403 trên live calls**

`IMBRACE_API_KEY` đã hết hạn hoặc sai. Tạo key mới:

```bash
curl -X POST https://app-gateway.dev.imbrace.co/private/backend/v1/third_party_token \
  -H "x-access-token: <your_existing_token>" \
  -H "Content-Type: application/json" \
  -d '{"expirationDays": 30}'
```
