Package `@imbrace/sdk` là client TypeScript/JavaScript chính thức cho Imbrace Gateway. Cung cấp quyền truy cập fully-typed vào tất cả các dịch vụ Imbrace cho cả server-side (Node.js) và client-side (browser).

### Tính Năng Nổi Bật

| Tính năng                | Chi tiết                                                             |
| ------------------------ | -------------------------------------------------------------------- |
| **Full Type Safety**     | 100% TypeScript — mọi request và response đều được type via `types/` |
| **Dual Authentication**  | API Key (server-to-server) hoặc Access Token (JWT / legacy token)    |
| **Auto Retry**           | 429 và 5xx tự động retry tối đa 2 lần với exponential backoff        |
| **Request Cancellation** | Hỗ trợ AbortSignal để hủy request đang chạy                          |
| **Multi-Environment**    | Preset `develop` / `sandbox` / `stable` với per-service URL override |
| **Streaming AI**         | AsyncGenerator-based streaming cho AI completions                    |

### Cách Hoạt Động

```
Your Code
   │
   ▼
ImbraceClient          ← entry point duy nhất, wire tất cả resources
   ├── TokenManager    ← lưu token thread-safe (OTP / JWT / API Key)
   └── HttpTransport   ← auth headers, retry, timeout, error mapping
         │
         ▼
   Imbrace Gateway     ← app-gatewayv2.imbrace.co (+ per-service URLs)
```

`ImbraceClient` tạo một `HttpTransport` và chia sẻ nó cho tất cả resource classes. Mỗi resource (`client.marketplace`, `client.platform`, ...) là wrapper mỏng, chỉ build URL đúng và gọi `transport.request()`.

### Kiến Trúc Nội Bộ

| Tầng             | File                    | Trách nhiệm                                        |
| ---------------- | ----------------------- | -------------------------------------------------- |
| Entry Point      | `client.ts`             | Khởi tạo tất cả resources, xử lý login helpers     |
| HTTP Transport   | `http.ts`               | Auth headers, retry logic, timeout, error mapping  |
| Token Manager    | `auth/token-manager.ts` | Lưu và lấy access token                            |
| Service Registry | `service-registry.ts`   | Resolve per-service URL từ environment preset      |
| Resources        | `resources/*.ts`        | API method wrapper cho từng domain (27 resources)  |
| Types            | `types/*.ts`            | TypeScript interfaces cho mọi API shape            |
| Errors           | `errors.ts`             | Định nghĩa `AuthError`, `ApiError`, `NetworkError` |

### Khi Nào Nên Dùng

| Kịch bản                                  | Auth khuyên dùng                            |
| ----------------------------------------- | ------------------------------------------- |
| Node.js backend, cron jobs, data pipeline | `apiKey` → header `x-api-key`               |
| Frontend sau khi user đăng nhập OTP/OAuth | `accessToken` JWT → `Authorization: Bearer` |
| Tích hợp legacy với opaque token          | `accessToken` opaque → `x-access-token`     |

:::tip
Với server-side, dùng `apiKey`. Với app user-facing, dùng OTP và để SDK tự quản lý token.
:::
