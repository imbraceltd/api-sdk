SDK hỗ trợ ba chiến lược xác thực. `HttpTransport` tự động chọn chiến lược phù hợp dựa trên những gì bạn truyền vào `ImbraceClient`.

### Các Chiến Lược Xác Thực

| Chiến lược       | Điều kiện kích hoạt                                          | Header gửi đi                                        |
| ---------------- | ------------------------------------------------------------ | ---------------------------------------------------- |
| **API Key**      | Option `apiKey` được truyền vào                              | `x-api-key: sk-xxx...`                               |
| **JWT Bearer**   | Token bắt đầu bằng `eyJ` và có 3 phần cách nhau bởi dấu chấm | `Authorization: Bearer eyJ...` + `x-organization-id` |
| **Legacy Token** | Access token bất kỳ khác                                     | `x-access-token: login_acc_...`                      |

:::note
Việc phát hiện JWT là tự động — transport kiểm tra định dạng token khi request. Bạn không cần cấu hình chiến lược thủ công.
:::

---

### API Key (Server-Side)

Dùng cho backend scripts, cron jobs, và server-to-server calls — credentials không bao giờ ra browser.

```typescript
import { ImbraceClient } from "@imbrace/sdk";

const client = new ImbraceClient({
  apiKey: process.env.IMBRACE_API_KEY,
  organizationId: process.env.IMBRACE_ORG_ID, // tùy chọn
});
```

Header `x-api-key` được gửi trên mọi request. Gateway resolve organization từ API key.

---

### Access Token (Client-Side)

Dùng sau khi user xác thực. Truyền token khi khởi tạo:

```typescript
// JWT (tự phát hiện bởi prefix eyJ...)
const client = new ImbraceClient({
  accessToken: "eyJhbGciOiJSUzI1NiJ9...",
  organizationId: "org_xxxxxxxx", // bắt buộc với JWT
});

// Legacy opaque token (tự phát hiện là non-JWT)
const client = new ImbraceClient({
  accessToken: "login_acc_xxxxxxxxxxxxx",
});
```

### Luồng OTP Login

SDK tích hợp sẵn luồng xác thực OTP. Đây là cách được khuyến nghị cho user-facing apps.

```typescript
import { ImbraceClient } from "@imbrace/sdk";

const client = new ImbraceClient();

// Bước 1: Gửi OTP đến email người dùng
await client.requestOtp("user@example.com");

// Bước 2: Người dùng nhập OTP — verify và lưu token tự động
await client.loginWithOtp("user@example.com", "123456");

// Bước 3: Tất cả các call tiếp theo đã được xác thực
const me = await client.platform.getMe();
```

Đăng nhập bằng email + password cũng được hỗ trợ:

```typescript
await client.login("user@example.com", "password123");
// Token được lưu vào TokenManager tự động
```

### Quản Lý Token

Sau khi xác thực, token sống trong `TokenManager`. Bạn có thể thao tác trực tiếp:

```typescript
// Thay token (ví dụ: sau khi OAuth refresh)
client.setAccessToken("new-eyJhbGci...");

// Xóa token (ví dụ: khi logout)
client.clearAccessToken();
```

---

### Multi-Tenant (Organization ID)

Khi dùng JWT Bearer, SDK tự động gửi `x-organization-id` trên mọi request:

```typescript
const client = new ImbraceClient({
  accessToken: jwtToken,
  organizationId: "org_xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
});
```

:::caution
`organizationId` chỉ được gửi khi dùng JWT Bearer. Với API Key, gateway tự resolve org từ key.
:::

### Thứ Tự Ưu Tiên Cấu Hình

Client merge config theo thứ tự (cao xuống thấp):

1. **Constructor options tường minh** — `new ImbraceClient({ apiKey: "..." })`
2. **Environment variables** — `IMBRACE_API_KEY`, `IMBRACE_GATEWAY_URL`
3. **Defaults** — `env: "stable"`, gateway `https://app-gateway.imbrace.co`
