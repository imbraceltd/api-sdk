SDK hỗ trợ hai loại thông tin xác thực. Truyền `apiKey` / `api_key` hoặc `accessToken` / `access_token` vào hàm tạo client — lớp vận chuyển (transport) tự động xử lý header.

### Nên dùng thông tin xác thực nào?

Lựa chọn phụ thuộc vào **vai trò của Imbrace trong sản phẩm của bạn**:

#### Build on Imbrace → dùng Access Token

Imbrace là backend của bạn. Người dùng cuối đăng nhập vào Imbrace (qua OTP hoặc mật khẩu); token truy cập `acc_...` của họ là thông tin xác thực SDK dùng cho mọi yêu cầu. Hệ thống xác thực, cơ sở dữ liệu và logic nghiệp vụ của Imbrace (assistant, knowledge hub, workflow, AI agent) đều chạy thay mặt cho người dùng đã đăng nhập.

Sản phẩm điển hình: widget chat, bảng điều khiển hoặc ứng dụng di động nơi danh tính mỗi người dùng là một người dùng Imbrace. Imbrace ghi lại lịch sử, quyền và nhật ký kiểm toán theo từng người dùng.

#### Wrap Imbrace → dùng API Key

Bạn có backend riêng, người dùng riêng, cơ sở dữ liệu riêng. Imbrace là một khả năng mà dịch vụ của bạn gọi đến — "dùng AI của Imbrace để trả lời" — cùng với mọi thứ khác bạn làm. Người dùng cuối không bao giờ thấy Imbrace; dịch vụ của bạn gọi Imbrace bằng một khóa `api_...` cấp tổ chức duy nhất (do quản trị viên org Imbrace cấp). Imbrace thấy một danh tính (người dùng dịch vụ của khóa API) và bạn tự xử lý việc gán theo người dùng ở phía mình.

Sản phẩm điển hình: CRM hiện có, hệ thống ticket hoặc công cụ nội bộ nhúng Imbrace như một tính năng. Một thông tin xác thực nằm trong tệp env của bạn và phục vụ mọi yêu cầu.

| | Build on Imbrace (Access Token) | Wrap Imbrace (API Key) |
| --- | --- | --- |
| **Ai chạy xác thực?** | Imbrace (đăng nhập OTP / mật khẩu) | Bạn (xác thực của bạn) |
| **Người dùng của ai?** | Người dùng Imbrace | Người dùng của bạn |
| **CSDL chính thức của ai?** | Imbrace | Của bạn |
| **Danh tính Imbrace thấy** | Người dùng cuối thực tế | Một tài khoản dịch vụ |
| **Gán theo người dùng upstream** | Được tích hợp sẵn | Tùy bạn |
| **Thời hạn sống của thông tin xác thực** | Theo phiên; làm mới khi cần | Dài hạn; không tự hết hạn |
| **Header được gửi** | `x-access-token: acc_...` | `x-api-key: api_...` |

> Cả hai loại thông tin xác thực đều là công dân hạng nhất. Hầu hết tài nguyên chấp nhận cả hai. Một số tính năng phụ thuộc vào ngữ cảnh người dùng (document artifact, lịch sử chat theo người dùng) chỉ có ý nghĩa khi dùng access token.

### Tham chiếu Header

| Thông tin xác thực | Header được gửi |
| --- | --- |
| `apiKey` / `api_key` | `x-api-key: api_xxx...` |
| `accessToken` / `access_token` | `x-access-token: acc_xxx...` |

Ngữ cảnh tổ chức được **mã hóa bên trong thông tin xác thực** — mọi khóa API và mọi token truy cập đều gắn với đúng một tổ chức. Gateway phân giải tổ chức trên mọi yêu cầu từ thông tin xác thực bạn đã gửi. Bạn không bao giờ truyền `organizationId` / `organization_id` cho SDK.

Để biết hướng dẫn thiết lập thông tin xác thực từng bước (biến môi trường, dotenv, secrets), xem [Hướng dẫn thiết lập](/getting-started/setup/#configure-credentials).

### Luồng đăng nhập OTP

Sử dụng luồng này để xác thực người dùng qua email OTP và lấy token phiên. SDK tự động lưu token sau khi đăng nhập.

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```typescript
    import { ImbraceClient } from "@imbrace/sdk"

    const client = new ImbraceClient({
      baseUrl: "https://app-gatewayv2.imbrace.co",
    })

    // Step 1: Send OTP to the user's email
    await client.requestOtp("user@example.com")

    // Step 2: User submits the OTP they received
    const loginRes = await client.loginWithOtp("user@example.com", "ABC123")
    // loginRes contains a short-lived login_acc_... token (stored internally)

    // Step 3: Exchange for a long-lived access token
    const { token, refresh_token } = await client.auth.exchangeAccessToken("org_your_org_id")

    // Step 4: Activate the token for all subsequent calls
    client.setAccessToken(token)

    // Now use any resource
    const { data: boards } = await client.boards.list()
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    from imbrace import ImbraceClient

    client = ImbraceClient()

    # Step 1: Send OTP to the user's email
    client.request_otp("user@example.com")

    # Step 2: User enters OTP — token stored automatically
    client.login_with_otp("user@example.com", "123456")

    # Step 3: All subsequent calls are authenticated
    me = client.platform.get_me()
    ```

    Biến thể async:

    ```python
    from imbrace import AsyncImbraceClient

    async with AsyncImbraceClient() as client:
        await client.request_otp("user@example.com")
        await client.login_with_otp("user@example.com", "123456")
        me = await client.platform.get_me()
    ```
  </TabItem>
</Tabs>

### Trình quản lý ngữ cảnh (chỉ Python)

Luôn bọc Python client trong trình quản lý ngữ cảnh để nhóm kết nối `httpx` bên dưới được đóng sạch sẽ:

```python
# Synchronous
with ImbraceClient(api_key="api_xxx") as client:
    me = client.platform.get_me()

# Asynchronous
async with AsyncImbraceClient(api_key="api_xxx") as client:
    me = await client.platform.get_me()
```
