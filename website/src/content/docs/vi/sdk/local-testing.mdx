Hướng dẫn này cho phép bạn kiểm thử **gói đã được xây dựng** giống hệt như người dùng sẽ cài đặt — không phải mã nguồn thô. Sử dụng trước khi phát hành phiên bản mới hoặc khi tái hiện lỗi mà người dùng báo cáo.

## Điều kiện tiên quyết

- TypeScript: Node 18+
- Python: Python 3.10+ và `pip`
- Thông tin AWS được cấu hình (hồ sơ `imbrace`) nếu bạn cần giải mã biến môi trường từ Ansible

## URL môi trường

Cùng một tập URL cơ sở gateway áp dụng cho cả hai SDK. Đặt giá trị này một lần trong môi trường kiểm thử của bạn:

| Môi trường | `IMBRACE_BASE_URL` / `IMBRACE_GATEWAY_URL` |
| ----------- | ------------------------------------------ |
| develop     | `https://app-gateway.dev.imbrace.co`       |
| sandbox     | `https://app-gateway.sandbox.imbrace.co`   |
| stable      | `https://app-gatewayv2.imbrace.co`         |

## Thông tin xác thực

Lấy thông tin xác thực trực tiếp từ Ansible (hoặc sao chép từ cổng thông tin):

```bash
# from the SDK repo root
AWS_PROFILE=imbrace sops -d ansible/dev/secrets.enc.env | grep IMBRACE >> /tmp/imbrace.env
```

Tối thiểu cần thiết cho các lệnh gọi trực tiếp:

| Biến | Nơi lấy |
| ----------------- | --------------------------------------------------------------------------------------------- |
| `IMBRACE_API_KEY` | Cổng thông tin Imbrace, hoặc `POST /private/backend/v1/third_party_token` với access token hiện có |
| `IMBRACE_BASE_URL` | Một trong các URL trong bảng trên (mặc định là dev nếu không được đặt)                              |

Ngữ cảnh tổ chức được mã hóa bên trong khóa API — bạn không cần truyền ID tổ chức.

## TypeScript — liên kết bản dist đã xây dựng

### Thiết lập một lần

```bash
cd ts
npm install
npm run build
npm link        # exposes @imbrace/sdk globally
```

Sau đó, trong thư mục kiểm thử:

```bash
cd ts/tests/local
npm link @imbrace/sdk
cp .env.example .env
# fill in .env
```

### Chạy kiểm thử

```bash
cd ts/tests/local
node test-local.mjs
```

- **Không có thông tin xác thực** — chỉ chạy kiểm tra khởi tạo + bề mặt tài nguyên (không có mạng).
- **Có `IMBRACE_API_KEY`** — chạy tất cả kiểm tra API trực tiếp với gateway trong `IMBRACE_BASE_URL`.

### Lặp lại các thay đổi SDK

Mỗi lần chỉnh sửa cần xây dựng lại để liên kết nhận được thay đổi:

```bash
# terminal 1 — ts/
npm run dev          # tsc --watch

# terminal 2 — ts/tests/local
node test-local.mjs  # re-run any time
```

## Python — cài đặt từ wheel hoặc editable

### Cài đặt editable (lặp nhanh nhất)

Từ thư mục `py/`:

```bash
cd py
pip install -e .
```

Các chỉnh sửa mã được nhận mà không cần cài đặt lại.

### Cài đặt wheel (xác minh hình dạng phát hành)

```bash
cd py
python -m build               # produces dist/imbrace-*.whl
pip install dist/imbrace-*.whl --force-reinstall
```

Cách này phát hiện các tệp bị thiếu, lỗi đóng gói và vấn đề đường dẫn import mà cài đặt editable che giấu.

### Chạy kiểm thử

```bash
cd py
pip install -r tests/requirements.txt
python -m pytest tests/
```

Đối với kiểm thử hồi quy toàn luồng tương ứng với [Hướng dẫn toàn luồng](/sdk/full-flow-guide/):

```bash
cd test-pip-pkg/py
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m pytest tests/test_guide_flow.py -v
```

Cùng kiểm thử đó chạy với cả `IMBRACE_API_KEY` và `IMBRACE_ACCESS_TOKEN` để bao phủ cả hai chế độ xác thực.

## Khắc phục sự cố

**`Cannot find package '@imbrace/sdk'`** (TypeScript)
Chạy lại `npm link @imbrace/sdk` từ bên trong thư mục kiểm thử. Liên kết có thể bị hỏng nếu bạn xóa và cài đặt lại `node_modules` trong `ts/`.

**`ERR_MODULE_NOT_FOUND` cho một tệp dist** (TypeScript)
Bản dựng chưa được chạy hoặc một tệp nguồn được thêm vào mà không xây dựng lại. Chạy `npm run build` trong `ts/`.

**`ModuleNotFoundError: No module named 'imbrace'`** (Python)
Gói chưa được cài đặt trong venv đang hoạt động. Chạy lại `pip install -e .` (editable) hoặc `pip install dist/imbrace-*.whl`.

**401 / 403 trên các lệnh gọi trực tiếp**
Thông tin xác thực của bạn đã hết hạn, bị thu hồi hoặc sai. Đối với khóa API, hãy tạo khóa mới:

```bash
curl -X POST https://app-gateway.dev.imbrace.co/private/backend/v1/third_party_token \
  -H "x-access-token: <your_existing_token>" \
  -H "Content-Type: application/json" \
  -d '{"expirationDays": 30}'
```

Đối với các lỗi runtime khác, xem [Xử lý lỗi](/sdk/error-handling/) và [Khắc phục sự cố](/guides/troubleshooting/).
